
Pendant ce TP, j’ai surtout travaillé sur les merges.
J’ai utilisé un merge sans fast-forward, ce qui crée un commit de merge visible dans l’historique.
Cette méthode permet de garder une trace claire de la fusion entre deux branches.

J’ai aussi revu les commandes importantes :
- git pull : met à jour ma branche et fusionne automatiquement.
- git fetch : récupère les nouveautés sans modifier ma branche.
- git revert : annule un commit proprement.
- git reset : revient en arrière localement en réécrivant l’historique.

Ce TP m’a permis de mieux comprendre comment organiser un projet et gérer des conflits.
